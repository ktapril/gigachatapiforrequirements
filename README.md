# сервис проверки студенческих работ

сервис для анализа текста студенческой работы на соответствие требованиям (введение, заключение, безличные конструкции, глаголы в прошедшем времени и др.) с использованием **GigaChat API**.

 **особенности:**  
 - полностью автономный запуск (не требует XAMPP, MySQL или сторонних баз данных)  
 - хранение пользователей в локальной SQLite-базе  
 - поддержка загрузки: `.txt`, `.docx`, `.pdf`  
 - ограничение количества проверок на пользователя  
 - возможность редактировать промпты под свои нужды

# требования

- PHP 8.1 или новее с расширениями: pdo_sqlite, zip, curl, mbstring
- Composer для установки зависимостей

# технологии

- PHP
- GigaChat API
- HTML / CSS / JS
- GitHub

# как установить:

1. склонируйте репозиторий
2. установите зависимости:
composer install
3. создайте папку для данных:
md storage
4. настройте API-ключ GigaChat
5. запустите встроенный сервер PHP:
php -S localhost:8000 -t public/

**как получить ключ:** 
1. перейдите на Sber Developer Studio: https://developers.sber.ru/
2. создайте проект и выберите GigaChat API
3. перейдите в «настроить API» и нажмите «получить новый ключ»
4. скопируйте Authorization Key (он выглядит как длинная строка в формате Base64)
5. вставьте его вместо текста в файле:
{
  "gigachat_auth_key": "aGVsbG86d29ybGQ="
}
6. сохраните как config/api_keys.json (без .example)

**как изменить промпты:** 
все промпты находятся в файле:
src/GigaChatClient.php

пример:
$prompt = "проверь, избегает ли текст безличных конструкций...";
вы можете редактировать любой промпт, чтобы адаптировать проверку под ваши требования

#структура проекта:
1. public/             # публичные файлы (входная точка index.php)
2. src/                # бизнес-логика (Auth, Database, GigaChatClient)
3. config/             # конфигурация (api_keys.json)
4. storage/            # данные (база users.db, загруженные файлы)
5. vendor/             # зависимости Composer 
6. composer.json       # список зависимостей
7. README.md           # этот файл
